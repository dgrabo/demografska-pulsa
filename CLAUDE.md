# Build Prompt: DemografskaPulsa — Croatian Demographic Data App

## Project context

You are building **DemografskaPulsa** — a free, publicly available civic tech web application that visualises demographic data for the Republic of Croatia using public sources (Croatian Bureau of Statistics / Državni zavod za statistiku). The goal is to make dry statistical data understandable to every citizen through an interactive map, charts, and projections.

The application is built **step by step**. Every step must be 100% functional and tested before moving on to the next. When you finish a step, wait for my approval and an explicit "continue to step X" instruction before proceeding.

---

## Tech stack

- **Frontend:** Next.js 14, JavaScript (not TypeScript), App Router
- **Styles:** CSS Modules + plain CSS (no Tailwind, no styled-components)
- **Map:** Leaflet.js + react-leaflet (interactive map of Croatia)
- **Charts:** Chart.js + react-chartjs-2
- **Animations:** Framer Motion (page transitions and card entrance only)
- **Data preprocessing:** Python 3, pandas, openpyxl
- **Hosting:** Vercel (free tier)
- **GeoJSON:** Croatian county and municipality boundaries (publicly available)

Do not use a database — everything runs on static JSON files generated by Python scripts from DZS Excel/CSV sources.

### STEP 2 — Python data processing script

**Goal:** A Python script that reads the original DZS Excel files and generates clean JSON files used by the application.

**Note:** The user must manually download these files from DZS and place them in `data-processing/raw/`:
- `popis_2021-stanovnistvo_po_gradovima_opcinama.xlsx` — direct link: `https://podaci.dzs.hr/media/td3jvrbu/popis_2021-stanovnistvo_po_gradovima_opcinama.xlsx`
- `Grad_01_HR.xls` (2011 census, population contingents by municipality) — direct link: `https://web.dzs.hr/Hrv/censuses/census2011/results/xls/Grad_01_HR.xls`

Create `data-processing/process_popis.py` that:

1. Reads both Excel files (2011 and 2021)
2. Normalises column names (trim, lowercase)
3. Matches municipalities between the two censuses by ID or name
4. Calculates:
   - `pad_apsolutni` = stanovnistvo_2021 − stanovnistvo_2011
   - `pad_postotak` = (pad_apsolutni / stanovnistvo_2011) × 100
   - `indeks_starenja` = (stanovnistvo_65plus / stanovnistvo_0_14) if available
5. Aggregates to county level
6. Exports two JSON files:
   - `../public/data/zupanije.json` (20+1 counties, aggregated data)
   - `../public/data/opcine.json` (all 556 towns/municipalities, detailed)
7. Prints summary statistics to the terminal (number of municipalities, max decline, min decline, etc.)

Also create `requirements.txt`:
```
pandas==2.1.0
openpyxl==3.1.2
```

**Step 2 completion check:** Script runs without errors, generates valid JSON files, summary stats are plausible (max decline ~−25%, min decline ~−1%).
